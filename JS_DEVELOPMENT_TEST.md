# JS Development Server Test

## Обзор

Этот workflow тестирует запуск JS development сервера и проверяет доступность всех необходимых файлов через curl.

## Файлы

- `.github/workflows/js-development-test.yml` - основной workflow файл
- `test-server-files.sh` - скрипт для фиксированного адреса (localhost:172.0.0.0)
- `test-server-files-flexible.sh` - гибкий скрипт для любого адреса

## Что тестируется

### 1. Запуск сервера
- Запускает `./gradlew :composeApp:jsBrowserDevelopmentRun`
- Ожидает запуска сервера (до 60 секунд)
- Автоматически определяет порт (8080, 3000, 8081, 8082, 8083)

### 2. Проверка файлов
- **HTML**: `index.html` - главная страница
- **JavaScript**: `composeApp.js` - основной JS файл
- **WASM**: `skiko.wasm`, `skikod8.wasm` - WebAssembly файлы
- **Модули**: `skiko.mjs`, `js-reexport-symbols.mjs`

### 3. Валидация содержимого
- Проверка HTML структуры
- Проверка JavaScript кода
- Проверка WASM магических байтов
- Проверка Content-Type заголовков

### 4. Производительность
- Измерение времени загрузки файлов
- Проверка размеров файлов

## Результаты тестирования

При успешном запуске сервера:

✅ **composeApp.js загружается** - основной JavaScript файл доступен  
✅ **WASM файлы загружаются** - WebAssembly модули работают  
✅ **HTML структура корректна** - страница отображается  
✅ **Сервер доступен** - все основные файлы обслуживаются  

⚠️ **Предупреждения**:
- Статические изображения могут быть недоступны (404) - это нормально для development режима
- Content-Type для .mjs файлов может отличаться от ожидаемого
- CORS заголовки отсутствуют (не критично для development)

## Локальный запуск

```bash
# Запуск сервера
./gradlew :composeApp:jsBrowserDevelopmentRun

# Тестирование (в другом терминале)
./test-server-files-flexible.sh http://localhost:8080
```

## Триггеры

Workflow запускается при:
- Pull Request в ветку `trunk`
- Push в ветку `trunk`  
- Ручной запуск через GitHub Actions UI

## Таймауты

- Общий таймаут: 20 минут
- Ожидание запуска сервера: до 60 секунд
- Таймаут подключения curl: 5 секунд

## Устранение неполадок

### Сервер не запускается
- Проверьте что порты 8080-8083 свободны
- Убедитесь что все зависимости установлены
- Проверьте логи Gradle

### Файлы не найдены (404)
- Убедитесь что сборка прошла успешно
- Проверьте что сервер запущен на правильном порту
- Проверьте пути к файлам в build/dist/

### WASM файлы не загружаются
- Проверьте что браузер поддерживает WebAssembly
- Убедитесь что сервер правильно обслуживает .wasm файлы
- Проверьте Content-Type: application/wasm
